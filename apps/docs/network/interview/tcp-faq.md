# TCP 高频问答

## 为什么要三次握手，不是两次？

关键句：三次握手同时解决“历史连接误建 + 双方序列号同步 + 双向收发能力确认”。

扩展说明：

1. 第三次 ACK 让服务端确认客户端已收到自己的 SYN+ACK。
2. 可以避免旧 SYN 延迟到达导致的伪连接。

## 为什么会有 TIME_WAIT？

关键句：TIME_WAIT 既保证最后 ACK 可靠到达，也让历史报文自然过期，避免污染新连接。

扩展说明：

1. 持续时间通常是 2MSL。
2. 如果最后 ACK 丢失，被动方会重发 FIN，主动方仍可应答。

## TCP 和 UDP 怎么选？

关键句：先看业务目标，追求可靠顺序选 TCP，追求低时延和可控策略选 UDP。

扩展说明：

1. TCP：可靠、有序、拥塞控制，适合文件传输、Web 请求。
2. UDP：无连接、低开销，适合实时音视频、DNS。

## 为什么会发生粘包？

关键句：TCP 是字节流，没有消息边界，应用层必须自己定义边界。

扩展说明：

1. 常见边界方式：固定长度、分隔符、长度字段。
2. RPC 框架通常通过自定义协议头解决。

## 快速重传和超时重传有什么区别？

关键句：超时重传靠时间阈值，快速重传靠重复 ACK 信号，后者响应更快。

扩展说明：

1. 超时重传更保守，但恢复慢。
2. 快速重传通常配合 SACK，能减少无效重传。
