# 应用层面试题

## HTTP 报文由哪些部分组成？

关键句：请求报文和响应报文都遵循“起始行 + 头部 + 空行 + 可选消息体”。

1. 请求报文：请求行、请求头、空行、请求体。
2. 响应报文：状态行、响应头、空行、响应体。

## 常见 HTTP 状态码怎么快速回答？

关键句：先按 1xx-5xx 分类，再补高频码。

1. 2xx：成功（200）。
2. 3xx：重定向（301 永久，302 临时，304 协商缓存命中）。
3. 4xx：客户端错误（400、401、403、404）。
4. 5xx：服务端错误（500、502、503、504）。

## 301 和 302 的区别是什么？

关键句：301 是永久迁移，302 是临时迁移。

1. 两者都会通过 `Location` 指示新地址。
2. 面试可补充：SEO 场景通常更关注 301。

## 502 和 504 的区别是什么？

关键句：都是网关错误，但 502 是“无效响应”，504 是“上游超时”。

1. 502 Bad Gateway：网关拿到了异常响应。
2. 504 Gateway Timeout：网关等待上游超时。

## GET 和 POST 有哪些核心差异？

关键句：语义不同，幂等性和缓存策略通常也不同。

1. GET 语义是获取资源，通常可缓存、可书签。
2. POST 语义是提交处理，通常不缓存、非幂等。
3. 实际项目里也要看接口设计，不能只背“参数在 URL / Body”。

## HTTP 长连接是什么？

关键句：复用同一条 TCP 连接处理多个请求，减少建连与挥手开销。

1. HTTP/1.0 默认短连接。
2. HTTP/1.1 默认开启 Keep-Alive。

## HTTP 为什么不安全？HTTPS 解决了什么？

关键句：HTTP 明文传输有窃听、篡改、冒充风险，HTTPS 通过 TLS 提供加密和身份认证。

1. HTTPS 在 TCP 和 HTTP 之间增加 TLS。
2. 通过证书校验服务端身份，通过会话密钥保证机密性和完整性。

## HTTPS 握手如何表达更稳？

关键句：客户端发能力，服务端回证书，协商密钥后进入加密通信。

1. `ClientHello`：版本、随机数、密码套件。
2. `ServerHello` + 证书：确认参数并给出证书。
3. 客户端验证证书并发送密钥交换材料。
4. 双方生成会话密钥并发送 `Finished`。

## HTTP/1.1、HTTP/2、HTTP/3 怎么对比？

关键句：1.1 文本协议，2 二进制多路复用，3 基于 QUIC 减轻队头阻塞。

1. HTTP/2：头部压缩、二进制分帧、多路复用、服务端推送。
2. HTTP/3：基于 UDP + QUIC，连接迁移更友好。

## DNS 解析流程怎么回答？

关键句：先本地缓存，再递归/迭代查询到权威 DNS。

1. 客户端问本地 DNS。
2. 本地 DNS 命中缓存则直接返回。
3. 未命中时按根域、顶级域、权威域逐级查询。
4. 返回 IP 并缓存结果。

## Cookie、Session、Token、JWT 怎么区分？

关键句：Cookie 是载体，Session 是服务端状态，Token/JWT 常用于无状态鉴权。

1. Cookie：保存在客户端，浏览器自动携带。
2. Session：保存在服务端，通常依赖 Cookie 中的会话标识。
3. Token/JWT：每次请求手动或统一中间件携带，适合分布式场景。
