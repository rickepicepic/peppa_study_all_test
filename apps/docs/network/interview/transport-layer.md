# 传输层面试题

## TCP 三次握手过程怎么说？

关键句：`SYN -> SYN+ACK -> ACK`，双方确认收发能力并同步初始序列号。

1. 客户端发 `SYN`，进入 `SYN-SENT`。
2. 服务端回 `SYN+ACK`，进入 `SYN-RCVD`。
3. 客户端回 `ACK` 后双方进入 `ESTABLISHED`。

## 为什么必须是三次，不是两次？

关键句：三次握手能避免历史连接误建，并确认双向可达。

1. 防止旧 `SYN` 延迟到达造成伪连接。
2. 两次握手无法让服务端确认客户端是否收到自己的响应。
3. 三次握手还能减少半连接资源浪费。

## 第三次握手 ACK 丢了会怎样？

关键句：客户端已建立，服务端会重传 `SYN+ACK`，直到超时放弃或收到 ACK。

1. ACK 本身一般不重传。
2. 对端通过重发对应报文触发补偿。

## TCP 四次挥手怎么讲？

关键句：连接是全双工，关闭发送和接收通道要分开确认。

1. 主动方发 `FIN`。
2. 被动方回 `ACK`，进入 `CLOSE_WAIT`。
3. 被动方处理完剩余数据后发 `FIN`。
4. 主动方回 `ACK` 并进入 `TIME_WAIT`。

## 为什么有 TIME_WAIT 和 2MSL？

关键句：保证最后 ACK 可重发，同时让旧报文自然过期。

1. 若最后 ACK 丢失，被动方会重发 `FIN`，主动方仍可响应。
2. 等待 2MSL 可以降低旧报文污染新连接的风险。

## TCP 和 UDP 的核心区别是什么？

关键句：TCP 重可靠有序，UDP 重轻量低时延。

1. TCP：面向连接、可靠传输、流控 + 拥塞控制。
2. UDP：无连接、无重传保证、开销小、实时性更好。

## TCP 为什么“可靠”？

关键句：依赖序列号、确认应答、超时重传、流量控制、拥塞控制共同实现。

1. 序列号保证顺序和去重。
2. ACK + 重传处理丢包。
3. 滑动窗口控制发送速率。
4. 拥塞算法避免网络雪崩。

## TCP 粘包怎么解决？

关键句：TCP 只有字节流没有消息边界，应用层必须定义边界。

常见方案：

1. 固定长度。
2. 分隔符。
3. 长度字段 + 消息体（最常见）。

## TCP 拥塞控制有哪些阶段？

关键句：慢启动、拥塞避免、快重传、快恢复。

1. 慢启动：指数增长，快速探测带宽。
2. 拥塞避免：线性增长，降低拥塞风险。
3. 快重传：收到重复 ACK 立即重传。
4. 快恢复：避免每次都回到初始窗口。
