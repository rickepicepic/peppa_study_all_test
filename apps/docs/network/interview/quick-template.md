# 3 分钟面试输出模板

## 模板一：回答原理题

1. 先定义：这个协议/机制是什么。
2. 再流程：按时序讲关键步骤。
3. 再目的：它解决了什么问题。
4. 再权衡：代价或局限是什么。

示例开头：

“TCP 三次握手本质是连接初始化阶段的状态同步，核心目标是同步序列号并确认双向收发能力。”

## 模板二：回答对比题

1. 给结论：A 更适合什么场景，B 更适合什么场景。
2. 给维度：连接方式、可靠性、时延、开销、适用业务。
3. 给边界：极端场景下会怎样。

示例开头：

“HTTP2 和 HTTP3 的核心差异在传输层，HTTP2 仍依赖 TCP，而 HTTP3 通过 QUIC 改善跨流阻塞问题。”

## 模板三：回答排障题

1. 明确现象（超时/慢/偶发失败）。
2. 分层排查（DNS -> 路由 -> 端口 -> 应用）。
3. 给证据（命令输出或监控指标）。
4. 给修复与复盘（短期止血 + 长期治理）。

## 临场建议

- 每题先用一句话给结论，再展开细节。
- 术语尽量配“它解决什么问题”。
- 避免纯背诵，补一句工程场景会更有说服力。
