# 分层模型与核心协议

## 常见分层

- OSI 七层：应用层、表示层、会话层、运输层、网络层、数据链路层、物理层。
- TCP/IP 四层：应用层、运输层、网络层、网络接口层。
- 工程常用五层：应用层、运输层、网络层、数据链路层、物理层。

## 核心理解

1. 分层是“职责划分”，不是“物理存在的墙”。
2. 上层依赖下层能力，下层对上层屏蔽实现细节。
3. 实际工程里，OSI 的表示层与会话层通常融合到应用层。

## 各层在做什么

### 应用层

- 目标：让应用可以表达业务语义。
- 典型协议：HTTP、DNS、SMTP、FTP、WebSocket。

### 运输层

- 目标：端到端传输。
- 典型协议：TCP（可靠、面向连接）、UDP（无连接、低开销）。

### 网络层

- 目标：跨网络寻址与转发。
- 典型协议：IP、ICMP。

### 数据链路层

- 目标：相邻节点帧传输、MAC 寻址、差错检测。
- 典型协议：以太网协议、ARP（常与链路层过程关联理解）。

### 物理层

- 目标：比特在物理介质中的编码与传输。

## 高频易混点

1. IP 不保证可靠、有序、无重复；这些通常由 TCP 提供。
2. HTTP 是应用层协议，HTTPS = HTTP + TLS。
3. ARP 解决“已知 IP，求下一跳 MAC”问题。
